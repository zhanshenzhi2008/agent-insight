## 部署方案总览

本项目提供**两套部署方案**，可根据实际场景选择：

| 方案 | 配置文件 | 适用场景 | 特点 |
|------|----------|----------|------|
| **方案 A：无 Traefix（默认）** | `docker-compose.yml` | 开发环境 / 内网 / 单机部署 | 端口直连，配置简单 |
| **方案 B：Traefix** | `docker-traefik/agent-insight/docker-compose.yml` | 生产环境 / 需要 HTTPS / 多服务 | 自动 HTTPS、负载均衡 |

> **关联文档**：[docker-traefik/README.md](../docker-traefik/README.md) — Traefix 部署方案详细说明

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 21+ | 后端运行环境，Spring Boot 4.1 |
| Node.js | 18+ | 前端构建，建议 20 LTS |
| Docker | 20.10+ | 容器化部署 |
| Docker Compose | 2.0+ | 多容器编排 |
| 磁盘空间 | 10GB+ | MySQL + MongoDB + Redis 数据持久化 |

## 2. 环境变量说明

所有配置通过环境变量注入，无需修改代码或配置文件。推荐使用 `.env` 文件集中管理。

### 2.1 数据库相关

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `MYSQL_HOST` | `localhost` | MySQL 主机地址（Docker Compose 内用 `mysql`） |
| `MYSQL_PORT` | `3306` | MySQL 端口 |
| `MYSQL_DB` | `agent_insight` | 数据库名 |
| `MYSQL_USERNAME` | `root` | 数据库用户名 |
| `MYSQL_PASSWORD` | `root830i` | 数据库密码 |
| `MONGODB_URI` | `mongodb://localhost:27017/agent_insight` | MongoDB 连接 URI（会覆盖 host/port） |
| `MONGODB_PORT` | `27017` | MongoDB 端口 |

### 2.2 Redis

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `REDIS_HOST` | `localhost` | Redis 主机地址（Docker Compose 内用 `redis`） |
| `REDIS_PORT` | `6379` | Redis 端口 |
| `REDIS_PWD` | _(空)_ | Redis 密码（无密码留空） |

### 2.3 应用配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `SERVER_PORT` | `9280` | 后端服务端口 |
| `INSIGHT_LOG_ROOT` | `/app/project/data` | 日志文件根目录（容器内路径） |
| `INSIGHT_SCRIPT_ROOT` | `/app/project/data/system/agt` | Agent 脚本根目录 |

### 2.4 AI 配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `AI_ENABLED` | `false` | AI 功能总开关 |
| `AI_PROVIDER` | `openai` | AI Provider（openai / deepseek / ollama） |
| `OPENAI_API_KEY` | _(空)_ | OpenAI API Key |
| `OPENAI_BASE_URL` | _(空)_ | OpenAI 代理地址（如 `https://api.xty.app/v1`） |
| `OPENAI_MODEL` | `gpt-4o` | OpenAI 模型名 |
| `DEEPSEEK_API_KEY` | _(空)_ | DeepSeek API Key |
| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com` | DeepSeek API 地址 |
| `DEEPSEEK_MODEL` | `deepseek-chat` | DeepSeek 模型名 |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama API 地址 |
| `OLLAMA_MODEL` | `qwen2.5` | Ollama 模型名 |

### 2.5 前端配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `VITE_API_BASE_URL` | `/api` | 前端 API 请求基础路径（Docker 中为 `/api`） |
| `WEB_PORT` | `80` | 前端服务端口（Docker Compose 映射到宿主机） |

### 2.6 示例 `.env` 文件

```bash
# 数据库
MYSQL_PASSWORD=root830i
MYSQL_DB=agent_insight
MYSQL_USERNAME=root

# Redis
REDIS_PWD=

# AI（按需配置）

# AI_ENABLED=true
# OPENAI_API_KEY=sk-xxx
# OPENAI_BASE_URL=https://api.xty.app/v1
```

## 3. 快速部署（Docker Compose）

### 3.1 拉取代码

```bash
git clone <repository-url>
cd agent-insight
```

### 3.2 配置环境变量

```bash
cp .env .env.local
# 编辑 .env.local，填入实际配置
```

### 3.3 构建并启动

```bash
# 前端构建（首次或代码变更后）
cd agent-insight-web
npm install
npm run build
cd ..

# 启动所有服务（后台运行）
docker-compose up -d

# 查看日志
docker-compose logs -f agent-insight-backend

# 查看所有服务状态
docker-compose ps
```

### 3.4 验证服务

| 服务 | 地址 | 验证命令 |
|------|------|----------|
| 前端 | http://localhost:80 | `curl http://localhost:80/health` |
| 后端 | http://localhost:9280 | `curl http://localhost:9280/actuator/health` |
| Swagger | http://localhost:9280/doc.html | 浏览器打开 |
| MySQL | localhost:3306 | `docker-compose exec mysql mysql -u llm_agent -p` |
| MongoDB | localhost:27017 | `docker-compose exec mongodb mongosh` |
| Redis | localhost:6379 | `docker-compose exec redis redis-cli` |

健康检查返回示例：

```bash
# 后端健康检查
$ curl http://localhost:9280/actuator/health
{"status":"UP"}
```

### 3.5 部署方案选择

#### 方案 A：无 Traefix（当前文档）

```
用户请求 ──→ 前端:80 ──→ 后端:9280
         ──→ 前端:80 ──→ nginx 代理 /api/* ──→ 后端:9280
```

- **配置文件**：`docker-compose.yml`
- **前端 Dockerfile**：`agent-insight-web/Dockerfile`
- **访问方式**：直接通过端口访问（http://服务器IP:80）
- **适合场景**：开发环境、内网部署、单台服务器

#### 方案 B：Traefix（参考 docker-traefik/README.md）

```
用户请求 ──→ Traefix:443 ──→ 前端容器:80
                             ──→ 后端容器:9280
```

- **配置文件**：`docker-traefik/agent-insight/docker-compose.yml`
- **前端 Dockerfile**：`agent-insight-web/Dockerfile`
- **访问方式**：通过域名 + Traefix 自动 HTTPS
- **适合场景**：生产环境、域名访问、需要 HTTPS

**快速切换到 Traefix 方案**：

```bash
# 1. 进入 Traefix 部署目录
cd docker-traefik/agent-insight

# 2. 配置环境变量
cp envs/db.env.example envs/db.env
cp envs/ai.env.example envs/ai.env
# 编辑 envs/*.env 填入实际配置

# 3. 启动
docker compose up -d

# 4. 配置域名 DNS 指向服务器 IP
```

## 4. 生产环境注意事项

### 4.1 安全建议

- **不要提交 `.env` 文件**：已加入 `.gitignore`，使用 `.env.example` 作为模板
- **数据库密码**：生产环境务必使用强密码，不要使用默认密码
- **AI API Key**：存储在安全的密钥管理服务（KMS/Vault），启动时注入环境变量
- **端口暴露**：生产环境只暴露必要端口（80/443），后端 9280 不直接对外暴露
- **HTTPS**：生产环境使用 Nginx/Caddy 反向代理到 80 端口，配置 TLS 证书

### 4.2 性能调优

- **JVM 内存**：根据服务器内存调整 `JAVA_OPTS`，建议 `-Xms512m -Xmx2g`
- **MySQL 连接池**：`maximum-pool-size` 根据并发量调整，默认 10
- **Redis**：`maxmemory` 根据缓存数据量设置，建议机器内存的 50%

### 4.3 日志管理

- 容器日志通过 `docker-compose logs` 查看
- 生产环境建议配置日志收集（ELK / Loki / CloudWatch）
- 日志级别在 `application.yml` 的 `logging.level.*` 中调整（当前仅维护单一配置，不做多 profile 切换）

## 6. 常见问题 FAQ

### Q1: 后端启动失败，报 `Connection refused: mysql:3306`

**原因**：MySQL 未就绪，后端先于 MySQL 启动。

**解决**：确保 `depends_on` + `healthcheck` 配置正确（已内置）。如果仍有问题，增加 `start_period` 时间。

```yaml
# docker-compose.yml 中的相关配置
agent-insight-backend:
  depends_on:
    mysql:
      condition: service_healthy
```

### Q2: MongoDB 连接失败，报 `Timeout waiting for a connection`

**原因**：MongoDB 连接池配置缺失，高并发时连接耗尽。

**解决**：已在 `DynamicDatasourceManager` 中添加连接池配置，生产环境可调整 `connectionPoolSize`。

### Q3: Redis 缓存未生效，查询响应时间无改善

**原因**：
1. Redis 服务未启动
2. 查询参数不完全一致（如 page=0 和 page=0.0 被视为不同）
3. 缓存 TTL 已过期

**排查**：

```bash
# 检查 Redis 是否正常运行
docker-compose exec redis redis-cli ping

# 查看 Redis 中的缓存 key
docker-compose exec redis redis-cli KEYS "query:*"

# 查看缓存详情
docker-compose exec redis redis-cli GET "query:<key>"
```

### Q4: 前端无法访问后端 API，报 502

**原因**：Nginx 反向代理配置错误。

**排查**：
1. 确认 `agent-insight-backend` 服务状态正常
2. 检查 `docker-compose logs agent-insight-frontend` 中的 nginx 错误日志
3. 确认 `Dockerfile.web` 中 `proxy_pass` 地址为 `http://agent-insight-backend:9280/`（注意末尾斜杠）

### Q5: 日志文件无法读取，提示"日志文件不存在"

**原因**：容器内日志路径与宿主机不一致。

**解决**：在 `docker-compose.yml` 中正确挂载日志目录：

```yaml
agent-insight-backend:
  volumes:
    - /your/host/logs:/app/project/data
  environment:
    INSIGHT_LOG_ROOT: /app/project/data
```

### Q6: AI 功能调用失败

**原因**：
1. `AI_ENABLED=true` 但 API Key 未配置
2. 网络不通（国内访问 OpenAI 需要代理）

**解决**：

```bash
# 确认环境变量
docker-compose exec agent-insight-backend env | grep AI

# 检查日志
docker-compose logs agent-insight-backend | grep -i ai
```

### Q7: 如何升级到新版本？

```bash
# 拉取新代码
git pull origin main

# 重新构建（仅后端变更）
docker-compose build agent-insight-backend
docker-compose up -d agent-insight-backend

# 前端也有变更
cd agent-insight-web && npm run build && cd ..
docker-compose build agent-insight-frontend
docker-compose up -d agent-insight-frontend

# 数据库迁移（如有）
# 执行数据库迁移脚本...
```
